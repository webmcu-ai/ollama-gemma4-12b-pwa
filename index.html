Ollama must be installed and running locally. This page calls localhost:11434 directly from your browser.

1
Download and install Ollama from ollama.com
2
Allow browser access. On Windows, open a command prompt and run:
setx OLLAMA_ORIGINS "null"
This is the safest setting — it allows this local HTML file to reach Ollama but blocks all internet pages from doing so. On Mac/Linux, add OLLAMA_ORIGINS="null" to your shell profile instead.
2b
If you install this page as a Chrome PWA (three-dot menu → Install), do this extra step — skip it if you are using the plain HTML file only.

After installing the PWA, open it, then open DevTools (F12) → Console tab → type location.origin and press Enter. You will see a string like chrome-extension://abcdef1234... — copy that full value.

Then run this in a command prompt, replacing the example ID with your real one:
setx OLLAMA_ORIGINS "null chrome-extension://YOUR-ID-HERE"
Note: this page cannot read its own PWA origin automatically until it is running as an installed PWA — the value shown below will say null while you are using the plain HTML file, which is correct. Once installed as a PWA the real ID will appear and you can copy it from here instead of DevTools.

3
Reboot your computer. The environment variable does not take effect until after a full restart.
4
Start Ollama (system tray on Windows). Return here — the status pill top-right will turn green when connected.
5
Pull a model using the field below, or run this in a terminal:
ollama pull gemma4:12b
Note: Ollama model tags use size shorthand — gemma4:2b, gemma4:12b (not e2b/e4b).
